# Transparenter — 画像透過デスクトップアプリ

AI に緑/青/白背景で出力させたキャラクター画像から、キャラクター部分だけをきれいに切り抜いて透過PNGにするためのアプリ。Tauri v2 製(macOS / Windows)。

## 使い方

1. `画像を開く` またはドラッグ&ドロップで画像を読み込む(背景色は四隅から自動推定され、即プレビューされる)
2. 推定が合っていなければ **スポイト** ツールで背景をクリック
3. スライダーで微調整
   - **類似度**: 透過する色の範囲。上げると背景の色ムラまで消える
   - **なめらかさ**: 縁のグラデーション幅
   - **色かぶり除去**: 輪郭や髪に染みた緑/青を抑制
   - **縁の収縮 / 縁の濃さ**: 輪郭に残るフリンジの処理
4. 取り切れない部分(髪の隙間の緑など)は **追加透過** ツールでクリックすると、その連結領域だけが透過される(⌘Z で取り消し)。「同じ色を画像全体から透過」にチェックすると連結でなく全体に効く
5. 体の一部が半透明になる等、色ベースで直せない部分は **ブラシ** ツールでドラッグして修正。「不透明に戻す / 透明にする」の2モード、サイズと硬さ(縁のぼかし)を調整可能。サイズは**画面上のpx指定**で、ズームインするほど画像上では細かく塗れる(画像上の直径も参考表示される)。1ストローク=1回の ⌘Z で取り消し。ブラシは最終調整の位置づけのため、ブラシ後にキー色や透過調整を変更しようとすると確認ダイアログが出て、OK するとブラシ編集だけが削除される(「半透明をなくす」はブラシ後も自由に変更可)。パネルの設定項目はツールに応じて出し分けられる
6. 半透明ピクセルを残したくない場合は **「半透明をなくす(0/1に確定)」** にチェック。しきい値以上のαは完全不透明、未満は完全透明に確定される(不透明化するピクセルは二値化前のαで背景色を除去してから昇格するので縁が緑残りしない)
7. `PNGを保存` でフル解像度の透過PNGを書き出し

表示切替: **結果 / 元画像 / アルファ / 半透明検出**(アルファ=αチャネルの白黒表示、半透明検出=**半透明ピクセルだけを赤**で警告表示)。パネルには半透明ピクセルの割合も常時表示される

ズーム: ピンチ or ⌘+ホイールで拡大縮小(カーソル位置基準)、スクロール or 中ボタンドラッグでパン、⌘+ / ⌘− / ⌘0(全体表示)。右下のズームコントロールからも操作できる

## なぜきれいに抜けるのか

よくある透過ツールの失敗(キャラ全体がうっすら透ける、髪の隙間に緑が残る)への対策として:

- **2段階しきい値**: キー色からの距離が `類似度` 以下なら完全透明、`類似度+なめらかさ` 以上なら**完全不透明**。キャラ本体に中途半端なαが付かない
- **適応的な色距離**: 緑/青背景では色差(CbCr)のみで判定。白背景ではキー彩度に応じて輝度も加味しつつ色差を増幅し、肌色などが「白に近い」と誤判定されない
- **decontamination**: 半透明ピクセルから `(色 − (1−α)·キー色) / α` で背景の混入成分を数学的に除去 → 髪の隙間の緑残りが消える
- **despill**: 不透明部分に染みた緑/青かぶりを抑制
- **スポット透過**: クリック地点の色に近い**連結領域だけ**をフラッドフィルで透過。すでに透明な領域は通り抜けないため、キャラ内の同系色(緑の瞳など)に穴が開かない

## 開発

```bash
npm install
npm run tauri dev     # 開発起動
npm run tauri build   # 配布用ビルド (src-tauri/target/release/bundle/)
npm run verify:core   # コアアルゴリズムの数値検証(合成画像で α と色を assert)
```

構成: 画像処理はすべてフロントエンド(`src/core/` の純関数 + Web Worker)。プレビューは長辺1200pxの縮小版でリアルタイム処理し、保存時のみフル解像度で処理して `fast-png` で直接エンコードする(Canvas の premultiplied alpha による色劣化を回避)。Rust側は dialog / fs プラグインの登録のみ。

```
src/core/     アルゴリズム(Tauri/DOM 非依存の純関数)
src/worker/   処理 Worker とメッセージ型
src/app/      UI・状態・I/O
scripts/      検証スクリプト(verify-core.ts, run-on-image.ts)
test-images/  テスト用画像
```
